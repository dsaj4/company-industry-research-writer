# Assets Guide

Use `assets/` for curated examples, not for dumping every report.

## Good asset types

- strong benchmark samples
- chapter-level before/after pairs
- high-quality figure anchor examples
- final approved report excerpts that show a reusable pattern

## Suggested subfolders

- `benchmark-samples/`
- `benchmark-results/`
- `chapter-pairs/`
- `style-anchors/`
- `figure-anchors/`

## Curation rule

Add an asset only if it teaches a reusable pattern.

Do not add:

- near-duplicate drafts
- low-quality experiments
- company-specific artifacts with no reusable writing lesson

## Naming suggestion

Use names that describe the lesson, for example:

- `benchmark-private-ai-report.md`
- `company-overview_before-after.md`
- `valuation-language_guardrails.md`
