# 质量评分卡

这份 reference 用于在句子级润色之前，先判断一份稿子是否已经具备继续打磨的基础。

## 先过三道 Gate

### Gate A：是否接近 benchmark 水平

检查：

- 结构是否完整
- 首页或开篇的判断密度是否足够
- 章节是否各司其职
- 证据和图表责任是否清楚
- 表达是否达到成熟研究稿 finish

### Gate B：是否明确是一份新报告

检查：

- 是否有独立 thesis
- 问题树是否围绕当前对象展开
- 是否存在 benchmark 套壳
- 证据是否属于当前对象，而不是借 benchmark 事实冒充

### Gate C：是否达到声明的篇幅与完成度

检查：

- 是否绑定 benchmark profile
- 总字数是否达到当前档位 `min`
- 所有核心章节是否达到各自 `min`
- 是否存在“前面展开、后面骨架化”
- 风险、竞争、商业化等关键章节是否被压缩到只剩提纲

Gate C 需要明确给出：

- `budget_attainment_rate`
- `chapter_floor_pass_rate`

若 Gate C 不通过，这份稿子不是“需要 polish”，而是“还没写够”。

## 加权维度

每项先按 1 到 5 分评分，再按权重折算。

| 维度 | 权重 |
| --- | --- |
| 接近 benchmark 程度 | 20 |
| 独立性与原创性 | 15 |
| 事实准确性与来源质量 | 20 |
| 论证强度 | 15 |
| 财务与估值措辞 | 10 |
| 图表与证据组织 | 10 |
| 可读性与表达 | 5 |
| 风险与边界意识 | 5 |

## 重点检查项

### 1. 接近 benchmark 程度

重点看：

- 结构是否完整
- 前页是否像压缩 thesis tree，而不是目录复述
- 章节是否写到位
- 预算和完成度是否足以支撑 `medium / full` 声明

### 2. 独立性与原创性

重点看：

- thesis 是否属于当前对象
- 问题设置是否属于当前对象
- 是否存在 benchmark shell drift

### 3. 事实准确性与来源质量

重点看：

- 核心主张是否有来源
- 名称、日期、产品、融资轮次等是否准确

### 4. 论证强度

重点看：

- 机制链是否清楚
- 判断是否被证据支撑
- 是否形成 `能力 -> 产品 -> 商业化 -> 经济含义` 或其他适配链条

### 5. 财务与估值措辞

重点看：

- 上市 / 非上市措辞是否正确
- 是否区分融资估值、目标估值、经营结果和收入线索
- 是否错误继承 benchmark 的评级或目标价逻辑

### 6. 图表与证据组织

重点看：

- 图表是否承担具体证据职责
- 图表和正文是否咬合
- 是否存在只堆图不解释

### 7. 可读性与表达

重点看：

- 是否 judgment-first
- 是否压缩
- 是否存在重复论证和空话

### 8. 风险与边界意识

重点看：

- 是否识别 thesis-breaking risk
- 是否承认不确定性
- 是否避免伪确定性和过度迁移

## 红线

以下任一命中，直接 block：

- 核心事实错误
- benchmark 套壳严重
- 上市 / 非上市措辞错误
- 估值类别混淆
- 核心结论无来源
- 图表和正文错配
- 声明 `medium / full`，但总字数和核心章节明显 underfill

## 问题优先级

### P0

- 事实错误
- 套壳风险
- 估值与财务措辞错误
- 核心结论无来源
- 严重 underfill

### P1

- thesis 太弱
- 章节逻辑断裂
- 关键证据缺失
- 章节与对象类型不匹配
- 关键章节低于 budget floor

### P2

- 段落组织一般
- 表达不够压缩
- 风险写法偏空

### P3

- 句子打磨
- 表层格式问题

## 最低 review 输出要求

至少输出：

- Gate A 结论
- Gate B 结论
- Gate C 结论
- 加权评分摘要
- 红线检查结果
- 预算审计结果
- `budget_attainment_rate`
- `chapter_floor_pass_rate`
- 3 到 7 条下一步修订动作
