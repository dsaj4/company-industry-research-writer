# Runs

Each report attempt should have its own run folder.

## Naming

Use:

`YYYYMMDD-HHMMSS_<topic>/`

Example:

`20260330-213000_moonshot-private-ai-report/`

## Recommended contents

- `00_request.md`
- `01_calibration.md`
- `02_thesis.md`
- `03_outline.md`
- `04_evidence_map.md`
- `05_draft.md`
- `06_review.md`
- `07_revision.md`
- `08_final.md`
- `09_upgrade_notes.md`

You can bootstrap a new run by copying:

`runs/_template/`

## If the report is chapter-orchestrated

Add:

- `00_master_outline.md`
- `01_assignment_board.md`
- `02_merged_report.md`
- `03_merge_notes.md`

And per chapter:

- `01_subchapter_breakdown.md`
- `02_chapter_draft.md`
- `03_chapter_comparison.md`
- `04_chapter_revision.md`

## Purpose

The run directory makes each writing cycle inspectable.

That is what allows future skill upgrades to be grounded in real diffs instead of vague impressions.
