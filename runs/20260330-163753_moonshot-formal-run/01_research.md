# Research

## 为什么这次必须先 research

本次对象命中 deeper research 的多个触发条件：

1. 月之暗面是非上市公司，融资与估值事实高度不稳定。
2. 报告会涉及用户位置、商业化、平台能力、生态信号等高风险分析语言。
3. 关键结论依赖多源确认，不能只靠单条媒体报道。
4. 当前 source pack 虽然足以启动正式 run，但还不足以直接写成 final 成稿。

## 当前已可视为“相对确定”的事实

1. 官方存在 `moonshot.cn`、`kimi.com` 与 `platform.moonshot.cn` 三类入口。
2. 开放平台已经形成 `kimi-k2.5 / kimi-k2 / moonshot-v1` 的分层定价体系。
3. 官方存在 `Contact Sales` 页面，说明企业客户支持流程已产品化。
4. 平台服务协议已公开，且涉及付费服务、客户数据与责任限制。
5. MoonshotAI 已建立官方 GitHub Organization。
6. OpenRouter 已分发 Kimi K2，说明其模型服务存在国际开发者侧触达。

## 当前可用但必须降级措辞的事实

1. `2025 年底完成 5 亿美元 C 轮融资、估值约 43 亿美元`
   - 可写为：`据 SCMP 报道，公司于 2025 年底完成新一轮融资，估值约 43 亿美元。`
2. `2026-03 寻求新融资、目标估值 180 亿美元`
   - 必须写为：`媒体报道称公司寻求新融资，目标估值约 180 亿美元。`
   - 不能写成：`公司估值 180 亿美元`
3. `QuestMobile 口径下处于 AI 原生应用头部梯队`
   - 可写为：`据 QuestMobile 口径，Kimi 处于 AI 原生应用头部梯队`
   - 不应直接写成：`Kimi 是绝对头部`
4. `Similarweb 的 AI referral`
   - 只能作为补充流量信号，不能等同于总流量或月活

## 当前最重要的未决问题

1. 月之暗面和 Kimi 的品牌与产品层级关系，正文如何清晰定义。
2. “头部用户梯队”是否还有第二个独立来源能交叉验证。
3. 商业化当前更应写成 `API + 企业服务 + Agent 扩展`，还是要以 `C 端产品心智 + B 端开放平台` 为主线。
4. OpenRouter 与 GitHub 信号在正文里应被当作“生态补充”还是“核心亮点”。
5. 竞争定位是否需要在首轮 outline 中就引入 DeepSeek、豆包、MiniMax 等对比对象。

## 建议继续补的 targeted research

1. 官方产品和模型页，补一轮产品结构与模型发布时间线。
