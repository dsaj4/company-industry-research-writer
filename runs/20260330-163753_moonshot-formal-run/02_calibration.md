原因：

- 需要借 `能力 -> 产品化 -> 商业化` 的桥接方式
- 需要借首页高密度核心结论和产品/平台证据写法

## 五个要模仿的点

1. 首页要先给出对象定义、赛道意义、差异化、商业化和风险边界。
2. 公司概况要先解释“月之暗面到底是什么类型的公司”，而不是先堆产品清单。
3. 亮点章节要把产品心智、模型能力和开发者商业化放进一个统一主线。
4. 商业化章节要明确区分已验证收入抓手与仍待验证的叙事空间。
5. 图表和截图要承担证据责任，而不是做页面装饰。

## 五个必须保持不同的点

1. 不继承 `MINIMAX` 的公股评级、目标价和完整预测壳。
2. 不把月之暗面直接写成消费流量公司，必须保留模型与平台属性。
3. 不把媒体口径的融资目标估值写成既成事实。
4. 不把 Similarweb 或 OpenRouter 的单点指标夸大成总量结论。
5. 不把 Moonshot 与 Kimi 的品牌关系写混。

## 本次 run 的边界提醒

- 这是非上市公司研究，估值部分默认写“估值边界/融资锚点”，不写目标价值。
- 研究对象的变化速度快，所有强判断都要保留置信度边界。
- 正文默认中文，不用英文分析 memo 的句法骨架。
