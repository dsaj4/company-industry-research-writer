# Company Overview Subchapter Breakdown

## Chapter problem

月之暗面的入口、模型、平台和专业助手是怎样的一体化骨架？

## Chapter intent

这一章不回答“公司有没有热度”，而是回答“这家公司到底靠什么把一个入口产品、一个模型供给层、一个平台层和一条专业助手路线串成同一套骨架”。写法上必须先把 `Kimi` 从公司全貌中拆出来，再说明它为什么又是最强入口；必须把平台与开发者触点写出来，但不能把触点直接夸大成经营结果；必须把 `Kimi+` 写成补线，而不是核心收入来源。

## Subchapter 1: Kimi is the front door, not the whole company

- Core judgment: `Kimi` 是 Moonshot 最显性的产品入口，但它只是公司骨架的最前端。
- What this subsection needs to prove:
  - 首页与产品页如何定义入口，而不是单一聊天框。
  - 首页上的网站、文档、表格、幻灯片、深度研究、Kimi Code、Kimi Claw、Agent Swarm Beta 等入口，说明产品已经从聊天扩展到工作流起点。
- Evidence plan:
  - `A01` Moonshot 官网首页
  - `A02` Kimi 产品首页
  - `A08` 官方 Playground 博客
  - `04_evidence_map.md` 中 `Company overview` 对应行
- Guardrails:
  - 只能写“入口和产品矩阵扩展”，不能写成“全公司已经被 Kimi 完全代表”。

## Subchapter 2: The model layer is the supply side of the company skeleton

- Core judgment: 模型层不是陪衬，而是 Moonshot 能把入口延伸成平台的供给底座。
- What this subsection needs to prove:
  - 平台页已明确展示 `kimi-k2.5 / kimi-k2 / kimi-k2-thinking` 等模型，以及分层价格体系。
  - 模型层已经被产品化为可定价、可调用、可区分层级的供给接口。
- Evidence plan:
  - `A03` 模型定价页
  - `A04` 开放平台首页
  - `04_evidence_map.md` 中定价和平台入口对应行
- Guardrails:
  - 只能写“分层供给已经形成”，不能写成“商业化规模已经证明”。

## Subchapter 3: The platform layer turns model power into callable interface

- Core judgment: Moonshot 的平台化不是额外装饰，而是把模型能力变成可接入、可服务、可销售接口的关键步骤。
- What this subsection needs to prove:
  - 官方平台首页和服务协议共同说明，平台已经不是单纯展示页，而是包含接入、合规和服务边界的产品层。
  - Contact Sales 的存在说明企业服务路径被显式产品化。
- Evidence plan:
  - `A04` 开放平台首页
  - `A06` Contact Sales
  - `A07` 开放平台服务协议
  - `A03` 模型定价页
  - `04_evidence_map.md` 中商业化边界相关行
- Guardrails:
  - 只能写“平台入口和合规边界已具备”，不能写成“平台收入已验证”。

## Subchapter 4: Developer touchpoints show ecosystem intent, not scale proof

- Core judgment: GitHub 组织、Playground 和官方博客，说明 Moonshot 已经在做开发者触点和工具链外延，但这些仍然只能作为生态意图和产品延展的证据。
- What this subsection needs to prove:
  - GitHub 组织页反映公开仓库和开发者触点。
  - Playground 博客展示了 Web Search、Memory、Excel、Code-Runner、Fetch、MCP 等工具调用场景。
  - 这些内容共同说明公司正在把模型能力外延到更复杂的工作流里。
- Evidence plan:
  - `A05` MoonshotAI GitHub Organization
  - `A08` Kimi Playground 官方博客
  - `A04` 开放平台首页
- Guardrails:
  - 只能写“开发者触点正在形成”，不能写成“生态已经成熟”。

## Subchapter 5: Kimi+ extends the assistant route, but it is still a supplementary layer

- Core judgment: `Kimi+` 说明 Moonshot 正在把普通助手路线延伸到专业助手和 Agent 路线，但它更适合作为产品路线补充，而不是这章的商业主结论。
- What this subsection needs to prove:
  - Kimi+ 的上线时间与产品定位，能说明专业助手并非临时包装，而是连续演化的一部分。
  - Kimi+ 应该被放在“专业助手补线”而不是“核心收入来源”。
- Evidence plan:
  - `B11` Kimi+ 相关文章
  - `A02` Kimi 产品首页
  - `A08` 官方 Playground 博客
  - `04_evidence_map.md` 中产品演化相关行
- Guardrails:
  - 只能写“专业助手路线补齐”，不能写成“核心变现已靠 Kimi+”。

## Subchapter 6: Closing judgment for this chapter

- Core judgment to land:
  - Moonshot 的公司骨架可以概括为 `入口 -> 模型 -> 平台 -> 专业助手` 四层联动。
  - 这四层已经在公开页面上形成了较完整的前台叙事，但真正能写进正文的，是“骨架已成、验证未满”的判断。
- Evidence pack to reuse in the draft:
  - `A01 A02 A03 A04 A05 A08 B11`
- Writing boundary:
  - 不写上市公司评级或目标价。
  - 不把产品前台、开发者触点和平台结构直接写成收入兑现。
  - 不把 `Kimi` 写成公司全貌，也不把 `Kimi+` 写成核心收入引擎。
