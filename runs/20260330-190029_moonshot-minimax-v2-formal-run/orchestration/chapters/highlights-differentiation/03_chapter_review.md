# Highlights / differentiation - Review

## Review conclusion

这一版已经把“用户入口、模型流通、工具扩展”三条主线写出来了，也明确保留了“势能形成中，不是壁垒已成”的边界。总体上可以进入修订，但还有几处需要收紧，不然会和公司概况、商业化章节发生轻微重叠。

## What works

1. 三条主线比较清楚，没有散成泛泛而谈的产品介绍。
2. `A02 / A04 / A05 / A08 / B11 / B15` 的证据链都用上了，且每条主线都有锚点。
3. 文字风格已经接近 MINIMAX 看点章节的写法，判断先行，证据跟随。
4. 对 `GitHub / OpenRouter / Playground` 的边界处理是对的，没有把生态信号写成收入结果。

## Issues to fix

### 1. 开头略长，判断可以更尖一点

现在第一段讲了三层主线，但还可以更快把“这章最值一看是什么”点出来。看点章节应该第一时间告诉读者：月之暗面的差异化在于入口、流通、任务执行三层联动，而不是先铺垫太多解释。

### 2. “工具调用 / MCP / 专业助手” 这一段里，MCP 术语可以更克制

当前版本提到了 `MCP`，但证据主要还是来自工具调用、Playground 和 Kimi+。如果不再额外加一处更直观的 `MCP` 证据，这里建议把术语稍微降级成“工具调用与连接能力”，避免术语超证据。

### 3. 第三方流通信号还可以再强调“信号”属性

`GitHub / OpenRouter` 这部分已经有边界，但还可以再多一句，明确它们只是“模型外溢的可见证据”，避免读者误解成生态已经成熟。

### 4. 与商业化章节的边界需要再压一遍

这一章不能提前把 `平台化` 写成商业化成果。当前文本已经基本守住，但有几句“开始成为平台”的表达，最好在修订里改成“开始具备平台化叙事”或“平台势能开始出现”。

## Review priorities

- P0: 缩短开头，强化判断句
- P1: 降低 `MCP` 术语浓度，确保证据匹配
- P2: 再压一遍生态信号和商业化边界
- P3: 适当压缩重复的“不是收入结果”表述，避免啰嗦
