# Revision Plan

## Objective

将当前完整 working draft 从“结构已成、论点已稳”推进到“达到 `MINIMAX full` 档位 floor 的正式长稿”。

## Revision order

1. 扩写公司概况
   - 为四层骨架各补 1 组更具体的页面或产品承接例子。
   - 增强“入口如何导向平台”的中间桥段，不只停留在结构判断。

2. 扩写赛道与竞争
   - 把 `DeepSeek / 豆包 / MiniMax` 三类压力各再压实一层。
   - 增加“为什么这会抬高月之暗面平台化门槛”的承接句，减少框架化复述。

3. 扩写商业化与估值边界
   - 补 1 到 2 个更硬的第三方采用或调用案例。
   - 继续保持“抓手已现、结果待证”的统一措辞，不把趋势信号写成收入结果。

4. 扩写风险
   - 把每条风险改成 `failure path` 写法：
     - 触发条件
     - 对 thesis 的影响
     - 为什么会放大估值压力

## Non-goals

- 不补股票评级、目标价、盈利预测表。
- 不把媒体报道中的目标估值写成当前估值。
- 不把官方生态展示页直接写成商业化已验证。
