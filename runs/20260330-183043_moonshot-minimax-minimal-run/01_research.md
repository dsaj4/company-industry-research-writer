## What this means for the Moonshot minimal run

### Can already be written

- 公司定位：
  - 不是单一应用，而是 `用户入口 + 模型能力 + 平台服务 + 助手扩展`
- 商业化抓手：
  - 官方定价
  - 联系销售
  - 平台协议
  - 第三方分发
- 生态信号：
  - GitHub
  - Playground / MCP
  - OpenRouter

### Should stay as boundaries

- 无法在本轮最小实现里写成硬结论的部分：
  - 绝对收入规模
  - 已验证的企业大客户结果
  - 可比估值与目标价格

## Minimal-implementation writing boundary

本轮可以把月之暗面写成：

- 以 `Kimi` 为用户入口
- 以模型和开放平台为能力底座
- 以 `Kimi+ / Playground / MCP / GitHub / OpenRouter` 作为产品与生态扩展信号
- 以融资与定价作为资本预期和商业化边界锚点

本轮不应把月之暗面写成：

- 已完成规模化收入验证的成熟平台公司
- 可直接套用上市公司 PS 估值的研究对象
- 只靠流量或单一应用增长驱动的产品公司
