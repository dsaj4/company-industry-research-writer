# Calibration

## Five things to imitate

1. 首页高密度核心观点，先给判断再给正文。
2. 稳定章节顺序：
   - 公司概况
   - 行业与竞争
   - 公司看点
   - 商业化与估值边界
   - 风险
3. 用 `模型/产品/平台/全球化或生态` 这样的能力链推进，不把公司写成单点产品介绍。
4. 每一章都要承担独立问题，不重复公司概况。
5. 图表和证据先规划再落文，哪怕最小实现不单独展示完整图表索引。

## Five things that must remain different

1. 不继承上市公司评级、目标价和 `PS` 壳。
2. 不继承 MiniMax 的产品矩阵、用户数据、收入预测和海外收入结构。
3. 不把月之暗面机械写成“已验证完成的上市 AI 平台公司”。
4. 不把拟议估值写成既成事实。
5. 不把第三方生态分发、流量和 GitHub 信号直接写成经营业绩。

## Minimal translation from MINIMAX to Moonshot

- `模型` 线：
  - 从 `全模态模型矩阵` 翻译为 `K2.5 / K2 / Moonshot-v1 的模型和平台层级`
- `产品` 线：
  - 从多产品矩阵翻译为 `Kimi 主入口 + Kimi+ 扩展`
- `平台` 线：
  - 从开放平台和 Coding Plan 翻译为 `官方定价 + 联系销售 + Playground / MCP`
- `全球化/生态` 线：
  - 从海外收入和出海产品翻译为 `OpenRouter / GitHub / 外部开发者生态`

## Copy-ready benchmark note

> 模仿 MINIMAX 时，月之暗面的最小实现应写成“入口已经建立、模型与平台框架已成、生态势能开始外溢，但商业化结果和估值兑现仍需继续验证”的中文研究稿，而不是上市股票推荐报告。
